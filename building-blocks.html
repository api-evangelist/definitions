These are the common building blocks I have pullled from my research so far.
Specification

The root specification for the API definition format, providing a base set of rules for all services and tools that employ the spec.
Generator

Generating an API definition format from an existing system or platform.
Parser

Parses a machine readable API definition and makes ready for use in specific language, structure, or platform.
Validator

Validates an API definition against its formal specification and schema, producing a valid or invalid response, with as much detail as possible.
Schema

Produces a specific schema, from an existing template, in a particular API definition format.
Converter

The conversion of an API definition from one format into another.
Database

Tooling that generates an API definition format from a database connection, allowing for API integration with common database formats.
Mock

Generating a mock interface for an API based upon the definition provided, allowing for testing, and playing with a copy of the API that is not for production.
Server

The auto generation of server side code from a machine readable API definition.
Client

The auto generation of client side code from a machine readable API definition format.
Single Page Application

Generating a single page applicaiton in a specific framework, derieved from an API definition, either as documentation for API, or as functional app interactive with API itself.
Command-Line

The usage of machine readable API definitions at the command line.
Powershell

The usage of machine readable API definitions via the powershell interface.
Testing

Generating, providing, and configuring API testing using machine readable API definition formats.
Monitoring

Setting up of API monitors to keep track of API operations over scheduled period of time using an API definition format.
Proxy

Generation and configuration of a proxy from a machine readable API definition.
Aggregator

Aggregation of APIs using a machine readable API definition format.
Editors

API definition editor, allowing for the creation, import, and export of API definition formats, providing a simple, IDE like API editing experience.
Documentation

Generation of static, or JavaScript interactive documentation from the API definition, helping developers understand what an API does.
Forms

Dynamically generating an HTML form from an API definition, allow for the structure, and handling to be driven by an API definition.
Real-Time

Establishing a connection with real-time functionality derived from an API definition format, allowing APIs to be connected with other real-time protocols.
Visualization

Generating a visualization from the meta data, or resources made available via a machine readable API definition.
